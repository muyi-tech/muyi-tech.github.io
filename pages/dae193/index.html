桥接模式的优势与限制

优势：

✅ 虚拟机与物理机在同一局域网

✅ 可以直接被同局域网其他设备访问

✅ 网络性能较好

✅ 支持各种网络协议

限制：

❌ 需要路由器有可用IP地址

❌ 在公共网络（如公司、学校）可能受限制

❌ 某些网络环境可能禁止

系统架构

在 Linux 系统中，可以通过以下命令快速判断系统架构是 amd64 (x86_64) 还是 arm64 (aarch64)：

使用 uname 命令（推荐）

uname -m

查看输出结果：

若显示 x86_64 → 系统为 AMD64（兼容 Intel/AMD 的 64 位架构）
若显示 aarch64 → 系统为 ARM64（如树莓派、苹果 M 系列芯片）

使用 lscpu 命令

lscpu | grep "Architecture"

x86_64 → AMD64
aarch64 → ARM64

安装 docker


# 更新系统
sudo dnf update

# 安装 Docker 所需的软件包和依赖项 
sudo dnf install -y yum-utils device-mapper-persistent-data lvm2

# 添加 Docker 官方的 YUM 仓库
sudo dnf config-manager --add-repo https://download.docker.com/linux/rhel/docker-ce.repo

# 阿里云，国内建议使用阿里云
sudo dnf config-manager --add-repo https://mirrors.aliyun.com/docker-ce/linux/rhel/docker-ce.repo


# 安装 Docker
sudo dnf -y install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

docker-ce               : This package provides the underlying technology for building and running docker containers (dockerd) 
docker-ce-cli           : Provides the command line interface (CLI) client docker tool (docker)
containerd.io           : Provides the container runtime (runc)
docker-buildx-plugin    : Docker Buildx plugin for the Docker CLI
docker-compose-plugin   : A plugin that provides the 'docker compose' subcommand


# 启动 Docker 并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 验证 Docker 版本
sudo docker --version

# 配置 Docker 加速器（国内）

sudo mkdir -p /etc/docker
sudo vim /etc/docker/daemon.json
# 填入一下内容，目前国内可以直接下载docker镜像，可用但不是很快。
{
    "registry-mirrors": [
        "https://docker.m.daocloud.io",
        "https://mirror.ccs.tencentyun.com",
        "https://docker.1ms.run",
        "https://dockerproxy.net"
    ]
}

# 重新加载 Docker 并重启服务
sudo systemctl daemon-reload
sudo systemctl restart docker

# 检测是否配置成功
docker info |grep -A1 Registry

# 升级版本
# 停掉docker服务
sudo systemctl stop docker 
# 删除旧版本   
sudo dnf remove -y docker-ce docker-ce-cli 

#重复本文的前5步，安装docker，并设置开机启动

Docker 服务开启 远程访问功能

vim /usr/lib/systemd/system/docker.service

# 在 ExecStart 后追加参数 -H tcp://0.0.0.0:2375 
ExecStart=/usr/bin/dockerd -H fd:// --containerd=/run/containerd/containerd.sock -H tcp://0.0.0.0:2375

# 重新加载 daemon 并重启 docker 服务
sudo systemctl daemon-reload
sudo systemctl restart docker

sudo systemctl status docker

# 检测daemon 服务连通性
curl 127.0.0.1:2375/info
docker -H tcp://127.0.0.1:2375 info

创建快照（至关重要！）

在操作之前，务必在虚拟机软件VMware中为这台虚拟机创建一个“快照”。

这样，如果使用中出现意外，您可以瞬间回滚到当前状态，确保数据安全。

虚机卡死必须强制关闭，宿主机突然断电，都可能会导致数据丢失，请务必创建快照。
