1. 什么是 Claude Code？

Claude Code 是 Anthropic 推出的终端 AI 编程代理（Agent）。与 IDE 插件（如 GitHub Copilot）不同，它不是一个代码补全工具——它是一个能理解整个代码库上下文、自主规划并执行多步开发任务的智能体。

核心能力：

  • 读/写文件、运行命令、调试错误
  • 理解项目结构，跨文件重构
  • 自主规划实施步骤，按计划执行
  • 通过 MCP 连接外部工具（GitHub、Slack、数据库等）
  • 支持并行多 Agent 协作
  • 可在 CI/CD 中以 headless 模式运行

2. 系统要求

项目要求
操作系统macOS 13.0+ / Windows 10 1809+ / Ubuntu 20.04+ / Debian 10+ / Alpine 3.19+
硬件4 GB+ RAM，x64 或 ARM64 处理器
网络需要互联网连接
ShellBash、Zsh、PowerShell 或 CMD
账号Anthropic Pro / Max / Team / Enterprise / Console 账号（免费版不支持 Claude Code）

3. 安装指南

3.1 原生安装（推荐）

原生安装自动在后台更新，是最省心的方式。

macOS / Linux / WSL：

curl -fsSL https://claude.ai/install.sh | bash

Windows PowerShell（管理员不需要）：

irm https://claude.ai/install.ps1 | iex

Windows CMD：

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

💡 提示： PowerShell 的提示符是 PS C:\>，CMD 是 C:\>。如果安装报错 syntax error near unexpected token '<'，说明你在错误的 shell 中运行了命令。

安装指定版本：

# 安装 stable 频道
curl -fsSL https://claude.ai/install.sh | bash -s stable

# 安装特定版本号
curl -fsSL https://claude.ai/install.sh | bash -s 2.1.89

3.2 Homebrew 安装（macOS）

# stable 频道（约晚一周，跳过有重大问题的版本）
brew install --cask claude-code

# latest 频道（即时更新）
brew install --cask claude-code@latest

⚠️ Homebrew 安装不会自动更新，需要手动运行 brew upgrade claude-code。

3.3 WinGet 安装（Windows）

winget install Anthropic.ClaudeCode

⚠️ 同样不会自动更新，定期运行 winget upgrade Anthropic.ClaudeCode。

3.4 npm 安装

npm install -g @anthropic-ai/claude-code

注意： 需要 Node.js 22+。不要用 sudo npm install -g！升级时用 npm install -g @anthropic-ai/claude-code@latest，不要用 npm update -g。

3.5 Linux 包管理器安装

apt（Debian / Ubuntu）：

sudo install -d -m 0755 /etc/apt/keyrings
sudo curl -fsSL https://downloads.claude.ai/keys/claude-code.asc \
  -o /etc/apt/keyrings/claude-code.asc
echo "deb [signed-by=/etc/apt/keyrings/claude-code.asc] https://downloads.claude.ai/claude-code/apt/stable stable main" \
  | sudo tee /etc/apt/sources.list.d/claude-code.list
sudo apt update
sudo apt install claude-code

dnf（Fedora / RHEL）：

sudo tee /etc/yum.repos.d/claude-code.repo <<'EOF'
[claude-code]
name=Claude Code
baseurl=https://downloads.claude.ai/claude-code/rpm/stable
enabled=1
gpgcheck=1
gpgkey=https://downloads.claude.ai/keys/claude-code.asc
EOF
sudo dnf install claude-code

4. 验证安装与诊断

# 查看版本
claude --version
# 输出示例：2.1.211 (Claude Code)

# 全面诊断
claude doctor

claude doctor 会检查安装健康状态、设置文件验证错误、以及任何警告和修复建议——而且只读，不会修改任何东西。

5. 认证与登录

首次运行 claude 会打开浏览器引导登录。你需要一个 Pro / Max / Team / Enterprise 或 Console 账号。

备选认证方式：

# 使用 API Key
export ANTHROPIC_API_KEY="sk-ant-xxxx"
claude

# 使用第三方提供商（Amazon Bedrock / Google Vertex AI / Microsoft Foundry）
# 参考官方文档配置

如果设置了 ANTHROPIC_API_KEY，Claude Code 会直接使用它而不打开浏览器。

5.1 🏠 配置第三方 API 提供商（国内用户/自建代理必看）

Claude Code 原生依赖 Anthropic 官方 API。在国内使用，或者想接入自建模型服务时，可以通过环境变量映射到任意兼容 OpenAI API 格式的提供商。

核心配置

# 1. 指向兼容 API 的地址（自建代理 / 第三方中转）
export ANTHROPIC_BASE_URL="https://your-proxy-or-provider.com/anthropic"
export ANTHROPIC_AUTH_TOKEN="your-api-key"

# 2. 模型映射（关键！）
export ANTHROPIC_MODEL="glm-5.1"                    # 兜底主模型
export ANTHROPIC_DEFAULT="deepseek-v4-flash"         # 默认模型

# 3. 三档模型映射（Claude Code 按场景自动选择）
export ANTHROPIC_DEFAULT_OPUS_MODEL="glm-5.1"              # OPUS 档 — 复杂推理/计划模式
export ANTHROPIC_DEFAULT_SONNET_MODEL="deepseek-v4-pro"    # SONNET 档 — 日常编码（主对话默认）
export ANTHROPIC_DEFAULT_HAIKU_MODEL="deepseek-v4-flash"   # HAIKU 档 — 轻量任务/摘要/压缩上下文

# 4. 子 Agent 模型（后台并行任务）
export CLAUDE_CODE_SUBAGENT_MODEL="deepseek-v4-flash"

各变量的含义与用途

环境变量作用建议映射模型使用场景
ANTHROPIC_BASE_URLAPI 端点地址你的代理或第三方服务 URL所有 API 请求的路由
ANTHROPIC_AUTH_TOKEN认证凭据该服务的 API Key替代官方 ANTHROPIC_API_KEY
ANTHROPIC_MODEL兜底主模型能力最强的模型没有匹配到三档时的 fallback
ANTHROPIC_DEFAULT默认模型均衡型模型未显式指定模型时的兜底
ANTHROPIC_DEFAULT_OPUS_MODELOPUS 档最强模型（如 glm-5.1）/plan 计划模式、复杂架构设计、深度推理
ANTHROPIC_DEFAULT_SONNET_MODELSONNET 档均衡模型（如 deepseek-v4-pro）日常编码、主对话默认、增量开发
ANTHROPIC_DEFAULT_HAIKU_MODELHAIKU 档轻量快速模型（如 deepseek-v4-flash）/compact 压缩上下文、摘要生成、简单问答
CLAUDE_CODE_SUBAGENT_MODEL子 Agent 模型性价比高的模型/batch 批量任务、后台并行 Agent

保存为持久配置

每次启动都手动 export 太麻烦，建议写到 shell 配置里：

# ~/.zshrc 或 ~/.bashrc
export ANTHROPIC_BASE_URL="https://your-proxy.com/anthropic"
export ANTHROPIC_AUTH_TOKEN="sk-your-key"

# 模型映射（按场景自动切换）
export ANTHROPIC_DEFAULT_OPUS_MODEL="glm-5.1"
export ANTHROPIC_DEFAULT_SONNET_MODEL="deepseek-v4-pro"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="deepseek-v4-flash"
export CLAUDE_CODE_SUBAGENT_MODEL="deepseek-v4-flash"

或者在 settings.json 的 env 块中配置：

// ~/.claude/settings.json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://your-proxy.com/anthropic",
    "ANTHROPIC_AUTH_TOKEN": "sk-your-key",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "glm-5.1",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "deepseek-v4-pro",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "deepseek-v4-flash",
    "CLAUDE_CODE_SUBAGENT_MODEL": "deepseek-v4-flash"
  }
}

实际效果

配置好后，Claude Code 会自动根据任务类型选择最合适的模型：

你输入: /plan "实现用户权限系统"
→ 自动选 OPUS 档 → glm-5.1（最强推理）

你输入: "给这个页面加一个搜索框"
→ 自动选 SONNET 档 → deepseek-v4-pro（日常开发）

Claude 内部: /compact
→ 自动选 HAIKU 档 → deepseek-v4-flash（轻量压缩）

💡 省钱的秘密： HAIKU 档负责高频低智商的 Token 消耗（摘要、压缩上下文），SONNET 档负责日常开发，OPUS 档只在你想让 Claude"好好想想"的时候启用。三个模型各司其职，浪费少了，效率高了。

6. 升级 Claude Code

6.1 自动更新

原生安装默认自动更新——在后台下载，下次启动时生效。

通过 /config 设置更新频道：

// ~/.claude/settings.json
{
  "autoUpdatesChannel": "latest"  // 或 "stable"
}

6.2 手动更新

claude update
# 输出：Successfully updated from 2.1.89 to version 2.1.211

6.3 切换发布频道

两个频道可选：

  • latest（默认）：第一时间收到新功能
  • stable：约晚一周，跳过有重大问题的版本

在交互会话中输入 /config → Auto-update channel 切换，或直接写 settings.json。

6.4 禁用自动更新

// ~/.claude/settings.json
{
  "env": {
    "DISABLE_AUTOUPDATER": "1"
  }
}

禁用后 claude update 仍然可用。想彻底阻止所有更新路径，用 DISABLE_UPDATES。

7. 卸载与完全重置

7.1 卸载程序

原生安装（macOS / Linux / WSL）：

rm -f ~/.local/bin/claude
rm -rf ~/.local/share/claude

原生安装（Windows PowerShell）：

Remove-Item -Path "$env:USERPROFILE\.local\bin\claude.exe" -Force
Remove-Item -Path "$env:USERPROFILE\.local\share\claude" -Recurse -Force

Homebrew：

brew uninstall --cask claude-code
# 或
brew uninstall --cask claude-code@latest

WinGet：

winget uninstall Anthropic.ClaudeCode

npm：

npm uninstall -g @anthropic-ai/claude-code

7.2 删除配置文件（完全重置）

⚠️ 这会删掉所有设置、允许的工具、MCP 配置和会话历史。

macOS / Linux / WSL：

# 用户级设置
rm -rf ~/.claude
rm ~/.claude.json

# 项目级设置（在项目目录下执行）
rm -rf .claude
rm -f .mcp.json

Windows PowerShell：

# 用户级设置
Remove-Item -Path "$env:USERPROFILE\.claude" -Recurse -Force
Remove-Item -Path "$env:USERPROFILE\.claude.json" -Force

# 项目级设置（在项目目录下执行）
Remove-Item -Path ".claude" -Recurse -Force
Remove-Item -Path ".mcp.json" -Force

7.3 重置模型设置

如果只是想重置模型相关的配置，不需要完全卸载：

方式一：通过交互命令

在 Claude Code 会话中输入：

/config

在设置界面中手动调整模型、Effort Level、Fast Mode 等设置。

方式二：编辑 settings.json 移除模型配置

// ~/.claude/settings.json — 移除或注释掉模型相关字段
{
  // 删除或注释掉下面这些行：
  // "model": "claude-sonnet-5",
  // "effortLevel": "xhigh",
  // "fastMode": true,
  // "advisorModel": "opus",
  // "availableModels": ["sonnet", "haiku"]
}

方式三：通过环境变量临时覆盖

# 启动时指定模型和 effort，不影响持久设置
claude --model claude-sonnet-5 --effort high

7.4 重置所有自定义配置的终极方案

如果你想要从零开始，彻底清除所有 Claude Code 的痕迹：

# 1. 卸载程序
# （根据你的安装方式选择对应命令）

# 2. 删除配置文件和缓存
rm -rf ~/.claude    # macOS/Linux
rm -rf ~/.claude.json

# 3. 检查是否有残留
which claude        # 应该返回 "not found"
claude --version    # 应该报错 command not found

# 4. 重新安装
curl -fsSL https://claude.ai/install.sh | bash
claude              # 首次运行重新登录

8. 常用功能案例详解

案例 1：初始化项目与 CLAUDE.md

场景： 你有一个新项目，想让 Claude Code 理解项目的架构、规范和约定。

步骤：

# 1. 进入项目目录
cd my-project

# 2. 启动 Claude Code
claude

# 3. 使用 /init 创建 CLAUDE.md
/init
# Claude 会分析项目结构并生成 CLAUDE.md 文件

手动编写 CLAUDE.md（推荐）：

# CLAUDE.md — 项目记忆

## 项目概述
这是一个基于 Vue 3 + TypeScript 的中后台管理系统。

## 目录结构
- src/ — 源代码
- src/components/ — 公共组件
- src/views/ — 页面视图
- src/api/ — API 接口层

## 编码规范
- 使用 Composition API 而非 Options API
- 组件名使用 PascalCase
- 变量和函数使用 camelCase
- 使用 TypeScript 严格模式

## 常用命令
- 构建：npm run build
- 开发：npm run dev
- 测试：npm run test
- 代码检查：npm run lint

## Git 规范
- 分支命名：feature/xxx, fix/xxx, refactor/xxx
- Commit 格式：type(scope): description

💡 CLAUDE.md 是 Claude Code 的"项目记忆卡"。每次启动都会自动读取它——能帮你节省 20-30% 的 Token！

案例 2：上下文管理 — /compact 与 /clear

场景： 对话上下文接近 200K 限制，Claude 开始"忘事"了。

/compact — 智能压缩（优先用这个）：

# 压缩上下文为摘要
/compact

# 压缩并保留关键信息
/compact 保留数据库 schema 和错误处理模式

什么时候用 /compact：

  • 上下文超过 100K token 时
  • Claude 的回答开始变得不准确
  • 你有很多历史对话但不想丢失关键上下文

/clear — 完全清空（换任务时用）：

# 彻底清空对话，从头开始
/clear
# 别名：/reset, /new

什么时候用 /clear：

  • 切换到完全不同的任务
  • 旧上下文会让 Claude 对当前任务产生偏见

💡 黄金法则： 换任务 → /clear；上下文太长但仍需保留关键信息 → /compact。

案例 3：模型切换与 Effort 控制

场景： 日常开发用 Sonnet 快跑，遇到难题切 Opus 深思考。

交互式切换：

# 进入模型选择器
/model

# 切换 Effort 级别
/effort low      # 简单任务，快速响应
/effort medium   # 日常开发（默认）
/effort high     # 复杂架构设计
/effort max      # Opus 4.6 专属，极限思考

# 切换快速模式
/fast on
/fast off

持久化设置：

// ~/.claude/settings.json
{
  "model": "claude-sonnet-5",
  "effortLevel": "high"
}

命令行指定（单次会话）：

claude --model claude-opus-4-6 --effort max

各订阅推荐的 Effort 策略：

订阅推荐模型默认 Effort复杂任务
Pro ($20)Sonnet 4.6mediumhigh
Max 5x ($100)Opus 4.6mediummax
TeamOpus 4.6mediummax
EnterpriseOpus 4.6highmax
Max 20x ($200)Opus 4.6maxmax

💡 Rails 模式（那其实是 Claude Code 的功能）：日常用 Sonnet，架构/调试切 Opus 4.6 + /effort max，做完再切回来。

案例 4：自定义 Slash 命令

场景： 你经常做同一件事——比如提交代码、review PR、运行测试——不想每次都输入一大段提示词。

创建自定义命令：

# 1. 创建命令目录（项目级）
mkdir -p .claude/commands

# 2. 创建 commit 命令
cat > .claude/commands/commit.md << 'EOF'
根据以下规范生成 git commit 信息：
- 格式: type(scope): description
- type: feat/fix/refactor/docs/style/test/chore
- scope: 影响的主要模块

然后帮我执行 git commit。
EOF

# 3. 创建 review 命令
cat > .claude/commands/review.md << 'EOF'
请对当前分支相比 main 的所有更改进行代码审查：
1. 检查逻辑错误和安全隐患
2. 检查是否遵循项目代码规范
3. 给出具体的修改建议
4. 列出需要在 PR 描述中包含的要点
EOF

使用自定义命令：

# 在 Claude Code 会话中
/commit "修复了用户登录时的 token 过期问题"
/review

用户级命令（跨项目共享）：

mkdir -p ~/.claude/commands
# 把 .md 文件放在 ~/.claude/commands/ 下即可在所有项目中可用

💡 写好一个 /commit 命令后，这辈子都不用手写 commit message 了。

案例 5：Hooks — 自动质量门禁

场景： 每次 Claude 写完代码自动跑 lint 和类型检查，不通过就不让改。

配置 Hooks：

// .claude/settings.json（项目级）
{
  "hooks": {
    "PreToolUse": {
      "Bash": {
        "command": "npm run lint:check",
        "on": "before"
      }
    },
    "PostToolUse": {
      "Write": {
        "command": "npm run type-check",
        "on": "after"
      },
      "Edit": {
        "command": "npm run type-check",
        "on": "after"
      }
    }
  }
}

更严格的场景——修改前端组件后跑测试：

{
  "hooks": {
    "PostToolUse": {
      "Edit": {
        "command": "cd {{cwd}} && npm run test:unit -- --related",
        "on": "after",
        "timeout": 30000
      }
    }
  }
}

💡 Hooks 是 Claude Code 最被低估的功能之一。用上它，你的代码质量会从"相信 Claude"变成"验证 Claude"。

案例 6：.claudeignore — 砍掉 Token 浪费

场景： 你的项目里有 node_modules、构建产物、图片资源等，Claude 不需要读它们。

创建 .claudeignore：

cat > .claudeignore << 'EOF'
# 构建输出
dist/
build/
.out/

# 依赖
node_modules/
vendor/
.pnp/

# 静态资源（Claude 不需要看图片/字体）
*.png
*.jpg
*.jpeg
*.gif
*.svg
*.ico
*.woff
*.woff2

# 日志和缓存
*.log
.cache/
.turbo/

# 环境文件
.env
.env.local
.env.*.local

# 版本控制
.git/
.gitattributes

# 大文件
*.zip
*.tar.gz
*.bin
EOF

💡 一个配置好的 .claudeignore 能减少 50-70% 的 Token 浪费！

案例 7：MCP 服务器集成

场景： 你想让 Claude 直接操作 GitHub、查询数据库、读取 Slack 消息。

添加 MCP 服务器：

案例 8：Headless 模式 — CI/CD 集成

场景： 你想在 GitHub Actions 中自动运行 Claude Code 做代码审查。

-p（print 模式）基本使用：

# 单次问答，非交互
claude -p "这个项目用到了哪些框架？" --print

# 带输出格式
claude -p "生成 CHANGELOG.md" --output-format json > changelog.json

# 带权限（完全非交互）
claude -p "修复 src/ 下所有 lint 错误" --print --allowed-tools "Bash,Read,Write,Edit"

GitHub Actions 集成示例：

# .github/workflows/claude-review.yml
name: Claude Code Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Get changed files
        id: changed
        run: |
          echo "files=$(git diff --name-only origin/main...HEAD | tr '\n' ' ')" >> $GITHUB_OUTPUT

      - name: Claude Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude -p "Review these modified files and identify potential issues: ${{ steps.changed.outputs.files }}" \
            --output-format json > review.json

      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = JSON.parse(fs.readFileSync('review.json', 'utf8'));
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## 🤖 Claude Code Review\n\n${review.result}`
            });

💡 Headless 模式让 Claude Code 从一个交互式终端工具变成了"可编程的 AI 命令行工具"。

10. 常见问题 FAQ

Q：Claude Code 和 VS Code 扩展有什么区别？
A：Claude Code 是一个独立的终端 Agent，不是编辑器插件。它有自己的 VS Code 扩展（管理集成用），但核心能力在终端中。claude 命令本身。

Q：免费 Claude 账号能用 Claude Code 吗？
A：不行。需要 Pro ($20/月)、Max、Team 或 Enterprise 账号。

Q：怎么重置所有自定义配置？
A：见第 7 节详细步骤——删 ~/.claude 和 ~/.claude.json 后重新安装。

Q：一次性重置模型设置，不删其他配置？
A：编辑 ~/.claude/settings.json，移除或注释掉 model、effortLevel、fastMode 等字段。下次启动会恢复默认。

Q：能不能让 Claude Code 输出中文？
A：在 ~/.claude/settings.json 中设置 "language": "chinese"，或直接在提示中说"用中文回答"。

Q：多个项目用不同配置怎么办？
A：利用"配置作用域"（scopes）：用户级（~/.claude/）放通用偏好，项目级（.claude/settings.json）提交到 Git 共享，本地级（.claude/settings.local.json）不提交。

Q：Token 用得太多怎么办？
A：配置 .claudeignore → 写简洁的 CLAUDE.md → 频繁 /compact → 日常用 Sonnet，必要时才切 Opus。

最后一句： Claude Code 不是一个"帮你写代码的工具"——它是你的结对编程伙伴，而且不用你出咖啡钱。花半天配置好，省下半辈子的时间。

本文基于 Claude Code v2.1.x 版本，官方文档为 code.claude.com/docs。
